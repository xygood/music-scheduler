## 二、排课信息（CX_JW_PKXX）

### 字段说明

| 字段代码 | 字段名称 | 数据来源 | 说明 |

| RKJSXM | 任课教师姓名 | result.teacherName | 直接使用排课结果中的教师姓名 |

### 重要规律

1. **周次合并显示**：同一教学班级、同一星期、同一节次的多个周次会合并为一条记录
2. **上课周次格式**：支持不连续周次，如 `第1-5、7-13、15-17周`、`第1周`、`第6-7、15周`
3. **ZC 和 SHOWZC 字段为空**：周次信息放在 SKZC 字段中
4. **减少行数**：通过周次合并，大大减少了排课信息表的行数

## 四、三个表的一致性

### 教学班级ID（JXBID）、教学班编号（JXBBH）、教学班级名称（JXBMC）

三个表中的这三个字段必须保持一致，使用相同的生成逻辑：

**JXBID/JXBBH 格式**：`课程编号 + 课程名称代码 + 序号`

**JXBMC 格式**：`课程名称 - 序号`

**示例**：
| 字段 | 值 |
|------|-----|
| JXBID | 108212435202QY2001 |
| JXBBH | 108212435202QY2001 |
| JXBMC | 器乐2-001 |

## 五、JXBID 生成规则

**格式**：`课程编号 + 课程名称代码 + 序号`

### 课程名称代码对照表

| 课程名称 | 课程名称代码 | 说明 |
|----------|--------------|------|
| 器乐1 | QY1 | QY=器乐拼音首字母 |
| 器乐2 | QY2 | |
| 器乐3 | QY3 | |
| 钢琴1 | GQ1 | GQ=钢琴拼音首字母 |
| 钢琴2 | GQ2 | |
| 钢琴4 | GQ4 | |
| 声乐1 | SY1 | SY=声乐拼音首字母 |
| 声乐2 | SY2 | |
| 声乐4* | SY4 | 带*号的课程也适用 |
| 中国器乐6* | QY6 | 中国器乐按器乐处理 |

### 序号规则

**按课程编号分别计数**：
- 同一课程编号从001开始递增
- 不同课程编号独立计数（即使课程名称相同）

**说明**：例如"钢琴2"可能有普通班和专升本两个不同的课程编号，每个课程编号都从001开始独立计数。

**示例**：
- 钢琴2（普通班，课程编号A）：001, 002, 003...
- 钢琴2（专升本，课程编号B）：001, 002, 003...
- 器乐2（普通班，课程编号C）：001, 002, 003...

### 生成示例

- 课程编号：`108212435202`
- 课程名称代码：`QY2`（器乐2）
- 序号：`001`（器乐类第1个）
- **JXBID**：`108212435202QY2001`

## 六、课程编号获取逻辑

### 直接使用排课结果中的课程编号

课程编号直接从排课结果列表（scheduleResults）的 courseId 字段获取，无需重新查找或计算。

**数据来源**：

```javascript
const courseNumber = result.courseId || '';
```

**说明**：
- scheduleResults 在计算时已经根据班级类型（普通班/专升本）匹配了正确的课程编号
- 导出时直接使用 result.courseId 即可

## 七、教师工号获取逻辑

### 从教师管理页面获取

教师工号从教师管理页面（availableTeachers）获取 `teacher_id` 字段。

**获取逻辑**：

const teacherInfo = availableTeachers.find(t => t.name === result.teacherName);
const teacherWorkId = teacherInfo?.teacher_id || '';

## 八、数据来源

### 排课结果列表（scheduleResults）

所有导出数据都直接从排课结果列表（scheduleResults）中提取，无需重新计算或分组。

**scheduleResults 包含的字段**：

| 字段名 | 说明 |
|--------|------|
| courseId | 课程编号 |
| courseName | 课程名称 |
| courseType | 课程类型（钢琴/声乐/器乐） |
| credits | 学分（1或2） |
| teacherName | 教师姓名 |
| groupSize | 小组人数 |
| students | 学生列表（包含学号、姓名、班级） |
| originalSchedules | 原始排课记录（包含周次、星期、节次、琴房信息） |
| room_id | 琴房编号 |
| room_name | 琴房名称 |

### 开课信息和学生选课数据

直接遍历 scheduleResults，每个小组生成一条记录。

**同一组的学生**：
- 共用同一个 JXBID
- 共用同一个教学班级名称
- 序号相同
- 开课信息中班级人数为 result.groupSize
- 开课信息中教学班组成为该组所有学生姓名

### 排课信息

**每个合并后的时间段生成一条记录**：遍历 scheduleResults，对于每个结果的 originalSchedules 数组，按星期和节次分组合并周次，每个合并后的时间段生成一条排课信息。

**周次合并规则**：
1. 按星期和节次分组
2. 同一组的时间段，收集所有周次
3. 去重并排序周次
4. 合并连续周次（如：1,2,3,4,5 → 1-5）
5. 生成周次字符串（如：第1-5、7-13、15-17周）

**示例**：
- 器乐2-001 第1-5、7-13、15-17周 周一 第2节（合并了多个周次）
- 器乐2-001 第1-16周 周一 第3节（连续周次合并）

这两条记录的 JXBID 相同，但上课时间不同。

## 九、上课周次格式

### 格式说明

上课周次（SKZC）字段支持以下格式：

1. **连续周次**：`第1-13周`、`第1-17周`
2. **单周**：`第1周`、`第6周`
3. **不连续周次**：`第1-5、7-13、15-17周`、`第6-7、15周`

### 示例

| SKZC | 说明 |
|------|------|
| 第1-5、7-13、15-17周 | 第1-5周、第7-13周、第15-17周（跳过第6、14周） |
| 第1周 | 仅第1周 |
| 第6-7、15周 | 第6-7周和第15周 |
| 第1-9、11-17周 | 第1-9周和第11-17周（跳过第10周） |

---

## 十、注意事项

1. **三个表一致性**：JXBID、JXBBH、JXBMC 三个字段在三个表中必须保持一致
2. **直接使用排课结果**：所有数据都直接从 scheduleResults 中提取，无需重新计算或分组
3. **课程编号**：直接使用 result.courseId，scheduleResults 已根据班级类型匹配
4. **学分**：直接使用 result.credits（1或2），从小组课列表中获取
5. **班级人数**：直接使用 result.groupSize，无需重新计算
6. **教师工号**：从教师管理页面的 `teacher_id` 字段获取，不是内部UUID
7. **学生学号**：从排课结果中学生列表的 `student_id` 字段获取，不是内部UUID
8. **序号按课程编号分别计数**：同一课程编号从001开始，不同课程编号独立计数（如：钢琴2普通班-001、钢琴2专升本-001）
9. **排课信息周次合并**：同一教学班级、同一星期、同一节次的多个周次会合并为一条记录，减少行数
10. **权限控制**：三个导出按钮只对管理员开放
