## 功能概述

数据备份功能提供以下核心能力：

| 功能 | 说明 | 权限 |
|------|------|------|
| 导出备份 | 将系统数据导出为 JSON 文件 | 仅管理员 |
| 恢复备份 | 从 JSON 文件恢复数据 | 仅管理员 |
| 清空数据 | 清空所有系统数据 | 仅管理员 |

---

## 访问入口

1. 使用管理员账户登录系统
2. 点击左侧导航菜单中的 **「数据备份与恢复」**
3. 进入备份管理页面

**管理员判断条件：**
- `faculty_id` 为 `ADMIN`
- `is_admin` 为 `true`
- `role` 为 `admin`
- `teacher_id` 为 `110`
- `email` 为 `admin@music.edu.cn`

## 导出备份

### 操作步骤

1. 在「导出备份」区域，选择要备份的数据类型

2. 可选的数据类型：

   | 数据类型 | 说明 |
   |----------|------|
   | 教师数据 | 所有教师信息 |
   | 学生数据 | 所有学生信息 |
   | 课程数据 | 所有课程信息 |
   | 教室数据 | 所有教室信息 |
   | 通适大课 | 通识大课排课数据 |
   | 专业大课排课结果 | 专业大课排课记录 |
   | 专业小课排课结果 | 专业小课排课记录 |
   | 用户账户 | 系统用户账户 |
   | 周次配置 | 学期周次配置 |
   | 班级数据 | 班级信息 |
   | 禁排时间 | 教师禁排时间段 |
   | 学生-教师分配 | 学生与教师的对应关系 |
   | 学生-专业分配 | 学生专业分配信息 |

3. （可选）输入自定义文件名

4. 点击 **「导出数据备份」** 按钮

5. 浏览器将自动下载 JSON 格式的备份文件

### 文件命名规则

```
默认格式：音乐排课系统备份_2026-02-26.json
自定义格式：自定义名称_2026-02-26.json
```

### 快捷操作

- **全选**：一键选中所有数据类型
- **全不选**：一键取消所有选择

## 恢复备份

### 操作步骤

1. 在「恢复备份」区域，点击 **「选择备份文件」** 按钮

2. 选择要恢复的 JSON 备份文件

3. 系统将自动验证文件格式并开始恢复

4. 恢复完成后，页面将自动刷新

### 恢复流程

```
读取备份文件 → 验证格式 → 恢复教师数据 → 恢复学生数据 
→ 恢复课程数据 → 恢复教室数据 → 恢复排课记录 
→ 恢复其他数据 → 刷新页面
```

### 数据转换说明

恢复过程中，系统会自动处理以下转换：

| 转换类型 | 说明 |
|----------|------|
| 教师ID转换 | 临时ID → 工号 |
| 学生ID转换 | UUID → 学号 |
| 姓名匹配 | 通过姓名查找对应的工号/学号 |

## 清空数据

### 操作步骤

1. 滚动到页面底部的「清空所有数据」区域

2. 点击 **「清空所有数据」** 按钮

3. 在确认对话框中，点击 **「确认清空」**

### 警告

⚠️ **此操作不可撤销！**

清空的数据包括：
- 教师数据
- 学生数据
- 课程数据
- 教室数据
- 排课记录
- 用户账户
- 周次配置
- 班级数据
- 禁排时间

**保留的数据：**
- 当前登录用户的会话信息

### 建议

在执行清空操作前，务必先导出当前数据的备份！

## Git 仓库操作

本项目使用 Git 进行版本控制，代码托管在 GitHub 上。

**仓库地址：** https://github.com/xygood/music-scheduler

### 首次获取项目

从 GitHub 克隆项目到本地：

```bash
# 克隆仓库（默认获取最新版本）
git clone https://github.com/xygood/music-scheduler.git

# 进入项目目录
cd music-scheduler
```

**克隆说明：**
- `git clone` 默认获取 main 分支的最新提交
- 克隆后会自动创建本地仓库和远程关联

### 克隆后启动项目

克隆完成后，按以下步骤启动项目：

#### 1. 安装前端依赖

```bash
# 在项目根目录执行
npm install
```

#### 2. 配置前端环境变量

```bash
# 复制环境变量模板
cp .env.example .env.development
```

编辑 `.env.development` 文件，配置后端地址：

```env
VITE_API_URL=http://localhost:5000
```

#### 3. 安装后端依赖

```bash
# 进入后端目录
cd server

# 安装 Python 依赖
pip install -r requirements.txt

# 或使用 pip3
pip3 install -r requirements.txt
```

#### 4. 配置后端环境变量

```bash
# 复制环境变量模板
cp .env.example .env
```

编辑 `.env` 文件，配置数据库等：

```env
DATABASE_URL=sqlite:///music_scheduler.db
SECRET_KEY=your-secret-key
FLASK_ENV=development
```

#### 5. 启动服务

**启动后端（终端1）：**
```bash
cd server
python3 app.py
```

后端启动成功后显示：
```
Starting server on port 5000 (env: development)
 * Running on http://127.0.0.1:5000
```

**启动前端（终端2）：**
```bash
npm run dev
```

前端启动成功后显示：
```
VITE v6.4.1  ready in xxx ms

➜  Local:   http://localhost:5173/
```

#### 6. 访问应用

浏览器打开 http://localhost:5173/ 即可使用系统。

#### 环境要求

| 环境 | 版本要求 |
|------|----------|
| Node.js | 18+ |
| Python | 3.8+ |
| npm | 9+ |

#### 端口说明

| 服务 | 默认端口 |
|------|----------|
| 前端 (Vite) | 5173 |
| 后端 (Flask) | 5000 |

如果端口被占用，可以：
- 前端：修改 `vite.config.ts` 中的 `server.port`
- 后端：设置环境变量 `PORT=5001`

### 推送代码到仓库

将本地代码推送到 GitHub：

```bash
# 1. 查看当前修改状态
git status

# 2. 添加所有修改的文件
git add .

# 3. 提交修改（附带提交信息）
git commit -m "feat: 添加新功能描述"

# 4. 推送到远程仓库
git push
```

**提交信息规范：**

| 类型 | 说明 | 示例 |
|------|------|------|
| `feat` | 新功能 | `feat: 添加用户登录功能` |
| `fix` | Bug 修复 | `fix: 修复登录验证问题` |
| `docs` | 文档更新 | `docs: 更新部署文档` |
| `style` | 代码格式 | `style: 格式化代码` |
| `refactor` | 重构 | `refactor: 重构排课逻辑` |
| `chore` | 构建/工具 | `chore: 更新依赖版本` |

### 从仓库获取最新代码

同步远程仓库的最新代码到本地：

```bash
# 拉取最新代码（推荐）
git pull

# 或者分步操作
git fetch        # 获取远程更新
git merge        # 合并到本地
```

**拉取说明：**
- `git pull` = `git fetch` + `git merge`
- 建议在推送前先拉取最新代码，避免冲突

### 处理合并冲突

当本地和远程有冲突时：

```bash
# 1. 拉取时发现冲突
git pull
# 提示：CONFLICT (content): Merge conflict in xxx

# 2. 查看冲突文件
git status

# 3. 选择保留版本
git checkout --ours <file>     # 保留本地版本
git checkout --theirs <file>   # 保留远程版本

# 4. 或手动编辑冲突文件，解决后
git add <file>
git commit -m "merge: 解决合并冲突"
git push
```

### 常用 Git 命令速查

| 命令 | 说明 |
|------|------|
| `git status` | 查看当前状态 |
| `git log --oneline -10` | 查看最近10次提交 |
| `git diff` | 查看未暂存的修改 |
| `git diff --cached` | 查看已暂存的修改 |
| `git branch` | 查看本地分支 |
| `git branch -a` | 查看所有分支（含远程） |
| `git checkout -b <分支名>` | 创建并切换到新分支 |
| `git remote -v` | 查看远程仓库地址 |

### 版本回退

如需回退到历史版本：

```bash
# 查看提交历史
git log --oneline

# 回退到指定版本（保留工作区修改）
git reset --soft <commit-hash>

# 回退到指定版本（丢弃所有修改）
git reset --hard <commit-hash>

# 强制推送到远程（谨慎使用）
git push -f
```

⚠️ **注意：** `git reset --hard` 会丢失未提交的修改，请谨慎使用！

## 注意事项

### 安全建议

1. **定期备份**：建议每周至少备份一次
2. **多地存储**：将备份文件保存到多个位置（本地、云盘、U盘等）
3. **验证备份**：定期尝试恢复备份文件，确保备份可用

### 操作建议

1. **恢复前备份**：在恢复备份前，先导出当前数据的备份
2. **重要操作前备份**：在进行大规模数据修改前，先创建备份
3. **保留历史备份**：不要只保留最新的备份，保留多个时间点的备份

### 常见问题

| 问题 | 解决方案 |
|------|----------|
| 恢复失败 | 检查备份文件格式是否正确，确保是系统导出的 JSON 文件 |
| 数据不完整 | 检查导出时是否选中了所有需要的数据类型 |
| 权限不足 | 确认使用管理员账户登录 |
