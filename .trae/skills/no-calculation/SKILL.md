# 禁止计算模式

## 触发条件

当用户发出以下类型的指令时，必须激活此技能：
- "不要计算"
- "不需要计算"
- "直接取值"
- "只取已有数据"
- "不要经过计算"

## 核心原则

**绝对禁止通过计算获得数据**，只能从以下来源直接提取：

1. **页面列表中已显示的数据**
   - 表格中可见的单元格值
   - 列表项中显示的文本
   - 统计卡片中展示的数字

2. **已存储的原始数据**
   - 数据库中的原始字段值
   - localStorage 中存储的值
   - API 返回的原始响应数据

3. **用户直接提供的数据**
   - 用户在表单中输入的值
   - 用户明确指定的数值

## 禁止的操作

以下操作在"禁止计算模式"下**绝对不允许**：

- ❌ 数值运算（加减乘除）
- ❌ 统计计算（求和、平均、计数）
- ❌ 百分比计算
- ❌ 时间差计算
- ❌ 数据转换计算
- ❌ 任何形式的派生数据计算

## 执行步骤

1. **识别数据需求**：明确用户需要什么数据
2. **定位数据来源**：在页面列表或存储中找到该数据
3. **直接提取**：原样获取已显示/已存储的值
4. **验证一致性**：确保提取的值与显示值完全一致
5. **报告来源**：告知用户数据的具体来源位置

## 示例场景

### 场景1：导出学生数据
用户指令："导出学生数据，不要计算"

正确做法：
- 直接从学生列表中提取每行显示的数据
- 不重新计算任何字段
- 不对数据进行转换或格式化

### 场景2：统计报表
用户指令："生成统计报表，直接取已有数据"

正确做法：
- 如果页面已显示"总计：50人"，直接使用50
- 不重新统计学生数量
- 不验证或校验已有数据

## 注意事项

- 如果所需数据在页面中不存在，应明确告知用户而非尝试计算
- 保持数据的原始格式，不做任何格式转换
- 如果数据有多个来源且值不一致，应询问用户使用哪个来源
