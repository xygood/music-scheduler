## 一、小组编组规则

### 1.1 人数限制

| 课程类型 | 最少人数 | 最多人数 |
|---------|---------|---------|
| 钢琴     | 2人     | 5人     |
| 声乐     | 2人     | 5人     |
| 器乐     | 2人     | 8人     |

**代码位置**: `validateGroup()` 函数 (约第3366行)

```typescript
const maxStudents = ['钢琴', '声乐'].includes(courseType) ? 5 : 8;
if (groupStudents.length > maxStudents) {
  return { valid: false, message: `${courseType}课每组最多${maxStudents}人` };
}
if (groupStudents.length < 2) {
  return { valid: false, message: `${courseType}课每组至少2人` };
}
```

### 1.2 主项/副项组合规则

#### 普通班（student_type = 'general'）

| 组合方式 | 主项人数 | 副项人数 | 说明 |
|---------|---------|---------|------|
| 方式1   | 2人     | 0人     | 最多2个主项，不能有副项 |
| 方式2   | 1人     | ≤2人    | 1个主项时最多2个副项 |
| 方式3   | 0人     | ≤4人    | 全部副项时最多4人（普通乐器） |
| 方式4   | 0人     | ≤8人    | 全部副项时最多8人（古筝、葫芦丝、竹笛） |

**代码位置**: `validateGroup()` 函数 (约第3394-3465行)

```typescript
// 普通班规则：
// 1. 最多2个主项+0个副项
// 2. 最多1个主项+2个副项
// 3. 最多0个主项+4个副项（葫芦丝、竹笛、古筝最多8人）
if (mainCourseStudents.length > 2) {
  return { valid: false, message: '普通班主项最多2人' };
}
if (mainCourseStudents.length === 2 && secondaryStudents.length > 0) {
  return { valid: false, message: '普通班2个主项时不能有副项' };
}
if (mainCourseStudents.length === 1 && secondaryStudents.length > 2) {
  return { valid: false, message: '普通班1个主项时最多2个副项' };
}
```

#### 专升本（student_type = 'upgrade'）

- 专升本学生**没有主项**，只有副项
- 副项最多 **2人**

**代码位置**: `validateGroup()` 函数 (约第3468-3482行)

```typescript
// 检查专升本副项人数限制
if (hasUpgrade) {
  if (secondaryStudents.length > 2) {
    return { valid: false, message: '专升本最多2个副项' };
  }
}
```

### 1.3 混组限制（不能混合编组）

| 限制类型 | 规则 | 错误提示 |

| 班级类型 | 普通班和专升本不能混合编组 | "普通班和专升本不能混合编组" |
| 年级 | 不同年级不能混合编组 | "不同年级不能混合编组" |
| 专业 | 不同专业不能混合编组 | "不同专业不能混合编组" |

## 二、排课冲突检测

### 2.1 冲突检测类型

| 冲突类型 | 检测内容 | 错误提示 |
|---------|---------|---------|
| 时段冲突 | 同一时间段已有课程安排 | "时段冲突：该时间段已有 X 门课程安排" |
| 教师冲突 | 教师在同一时间段已有其他课程 | "教师时间冲突：您在周X第X节已有 XXX 课程" |
| 学生冲突 | 学生在同一时间段已有其他课程 | "学生冲突：XXX 在周X第X节已有 XXX 课程" |
| 琴房冲突 | 琴房在同一时间段已被其他教师占用 | "琴房冲突：XXX 在周X第X节已被 XXX 占用" |
| 禁排时段 | 该时间段为禁排时间 | "禁排时段冲突：该时间段为禁排时间" |
| 周次范围 | 课程周次范围与已有课程重叠 | "课程周次冲突：该时间段与已有课程周次范围重叠" |
| 大课表冲突 | 与大课表时间冲突 | "大课表冲突：该时间段与大学大课表时间冲突" |

**代码位置**: `handleDrop()` 函数 (约第4446-4700行)

### 2.2 冲突检测顺序

1. 验证分组有效性
2. 检查时段冲突（已有课程）
3. 获取/创建课程
4. 检查琴房冲突
5. 检查学生冲突（跨教师）
6. 检查教师冲突（跨教师）
7. 检查禁排时段冲突
8. 检查周次范围冲突
9. 检查大课表冲突
10. 检查课时是否已排满
11. 创建排课记录

## 三、禁排时间逻辑

### 3.1 禁排类型

| 类型 | 说明 |
|-----|------|
| 全周禁排 | 整周都不能排课 |
| 部分禁排 | 特定周次的特定天/节次不能排课 |
| 班级关联禁排 | 只对特定班级生效的禁排 |

### 3.2 禁排检测函数

- `isWeekFullyBlocked()`: 检查周次是否为全周禁排
- `hasWeekPartialBlock()`: 检查周次是否有部分禁排
- `isWeekBlocked()`: 检查周次是否在禁排范围内
- `checkWeekBlockedStatus()`: 使用专业大课禁排数据检查
- `checkBlockedSlotConflict()`: 检查禁排时段冲突

**代码位置**: 约第351-410行

### 3.3 禁排数据来源

1. **blockedSlotService**: 系统配置的禁排时段
2. **localStorage 'music_scheduler_imported_blocked_times'**: 导入的专业大课禁排数据

## 四、批量选择功能

### 4.1 选择模式

| 模式 | 说明 |
|-----|------|
| 单次选择 | 每次点击选择一个周次 |
| 批量选择 | 自动填充可用周次直到选满16节 |
| 范围选择 | 拖拽选择多个时段 |

### 4.2 批量选择逻辑

**课时要求**: 专业小课固定 **16课时**（16节次）

**自动填充规则**:
1. 从当前点击的周次开始
2. 自动跳过禁排周次
3. 自动跳过已选择的周次
4. 循环选择直到选满16节或遍历完所有可用周次

**代码位置**: 约第1194-1270行

```typescript
// 批量选择模式
const requiredWeeks = 16;  // 固定16课时
const remainingWeeks = Math.max(0, requiredWeeks - totalSelectedCount);

// 自动填充可用周次（自动跳过禁排周次）
for (let w = 1; w <= totalWeeks; w++) {
  if (alreadySelectedWeeks.has(w)) continue;
  if (isSlotAvailableSync(day, period, w)) {
    availableWeeks.push(w);
  }
}
```

### 4.3 时段可用性检查 `isSlotAvailableSync()`

检查内容:
1. 全周禁排周次检查
2. 专业大课禁排时间数据检查
3. 教师已排定时间检查
4. 教师专业大课/理论课检查

**代码位置**: 约第910-1100行

### 5.1 进度计算规则

- **总课时**: 固定 16 课时
- **已完成课时**: 根据排课记录计算（去重后的时间槽数量）
- **剩余课时**: 总课时 - 已完成课时

**代码位置**: `studentProgress` useMemo (约第4195-4285行)

```typescript
const required = 16; // 固定16课时
const completed = uniqueTimeSlots.size;
const remaining = Math.max(0, required - completed);
```

## 九、学生来源标识

### 9.1 studentSources Map

用于标识学生是作为"主项学生"还是"副项学生"加入小组：

| 来源 | 说明 |
|-----|------|
| 'primary' | 主项学生（教师是学生的主项教师） |
| 'secondary' | 副项学生（教师是学生的副项教师） |

**代码位置**: 约第2196-2231行

### 9.2 来源判断逻辑

```typescript
// 检查是否为主项教师
const isPrimaryTeacher = student.teacher_id === teacher.id || 
                         student.primary_teacher_id === teacher.id;

// 检查是否为副项教师
const isSecondaryTeacher = student.secondary_instruments?.some(...);

// 根据匹配结果设置来源
if (isPrimaryTeacher) {
  studentSources.set(student.id, 'primary');
} else if (isSecondaryTeacher) {
  studentSources.set(student.id, 'secondary');
}
```

## 十、关键数据结构

### 10.1 Student 类型关键字段

```typescript
interface Student {
  id: string;
  name: string;
  major_class: string;           // 班级名称，如"音乐学2303"
  student_type: 'general' | 'upgrade';  // 学生类型
  primary_instrument: string;    // 主项乐器
  secondary_instruments: string[]; // 副项乐器列表
  teacher_id: string;            // 主项教师ID
  secondary_teacher1_id?: string; // 副项1教师ID
  secondary_teacher2_id?: string; // 副项2教师ID
  secondary_teacher3_id?: string; // 副项3教师ID
  __courseType?: string;         // 当前专业类型（添加到小组时设置）
}
```

```typescript
interface ScheduledClass {
  id: string;
  teacher_id: string;

  course_name: string;
```
